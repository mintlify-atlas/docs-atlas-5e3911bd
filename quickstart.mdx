# Quickstart

This guide will help you set up and run the AI Basketball Referee system on your local machine. You'll be detecting basketball violations in real-time within minutes.

## Prerequisites

Before you begin, ensure you have:

- Python 3.11 or higher
- A webcam or basketball game video file
- Git installed on your system
- Conda or another Python environment manager

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the AI Basketball Referee project to your local machine:

    ```bash
    git clone <repository-url>
    cd ai-basketball-referee
    ```
  </Step>

  <Step title="Create a conda environment">
    Create and activate a new conda environment with Python 3.11:

    ```bash
    conda create -n exercise-tracking python=3.11
    conda activate exercise-tracking
    ```

    <Note>
      The environment is named `exercise-tracking` to match the original project setup, but you can name it whatever you prefer.
    </Note>
  </Step>

  <Step title="Install dependencies">
    Install the Ultralytics package, which provides the YOLO models:

    ```bash
    pip install ultralytics
    ```

    For additional features like audio feedback, you may also want to install:

    ```bash
    pip install gtts playsound
    ```
  </Step>

  <Step title="Download the basketball detection model">
    The custom basketball detection model is too large for GitHub. Download it from Google Drive:

    [Download basketballModel.pt](https://drive.google.com/file/d/1e6HLRuhh1IEmxOFaxHQMxfRqhzD92t3B/view?usp=sharing)

    Place the downloaded `basketballModel.pt` file in the root directory of the project.

    <Warning>
      The system will not work without this model file. Make sure to download it before proceeding.
    </Warning>
  </Step>
</Steps>

## Running the system

The project includes several Python scripts for different violation detection scenarios. The two main scripts for real-time referee calls are:

### Travel detection

Detect traveling violations when players take steps without dribbling:

<CodeGroup>
```bash Webcam input
python travel_detection.py
```

```python Modify for video file
# In travel_detection.py, change line 16:
cap = cv2.VideoCapture("path/to/your/video.mp4")
```
</CodeGroup>

The script will:
- Open your webcam or video file
- Display the video feed with pose estimation and ball detection overlays
- Show dribble count in real-time
- Apply a blue tint and display "Travel Detected!" when a violation occurs
- Save violation footage to the `travel_footage/` directory

<Info>
  Press `q` to quit the video display at any time.
</Info>

### Double dribble detection

Detect double dribble violations when players dribble after holding the ball:

<CodeGroup>
```bash Webcam input
python double_dribble.py
```

```python Modify for video file
# In double_dribble.py, change line 19:
self.cap = cv2.VideoCapture("path/to/your/video.mp4")
```
</CodeGroup>

The script provides:
- Real-time ball and wrist tracking
- Visual indicators when the ball is being held (blue tint)
- Red tint and "Double dribble!" message when a violation is detected
- Debug information including ball coordinates, wrist positions, and dribble count

## Understanding the output

### Visual indicators

The system uses color-coded overlays to communicate violations:

| Color | Meaning |
|-------|--------|
| Blue tint | Ball is being held (double dribble detection) or travel detected |
| Red tint | Double dribble violation detected |
| Green box | Basketball detection bounding box |
| Skeleton overlay | Player pose estimation keypoints |

### Console output

You'll see real-time console output showing:

```bash
Step taken: 1
Step taken: 2
Travel detected!
```

Or for double dribbles:

```bash
Ball coordinates: (x=532.12, y=487.34)
The ball is being held.
Double dribble!
```

### Saved footage

When a travel is detected, the system automatically saves video clips to `travel_footage/` with timestamps:

```bash
travel_footage/
├── travel_20240315-143022.mp4
├── travel_20240315-143845.mp4
└── travel_20240315-144512.mp4
```

Each clip includes 30 frames before and 60 frames after the violation for complete context.

## Advanced configuration

You can customize the detection sensitivity by modifying key parameters in the Python scripts.

### Travel detection parameters

```python travel_detection.py
# Line 28-30: Adjust these thresholds
dribble_threshold = 18  # Vertical ball movement to count as dribble
step_threshold = 5      # Ankle displacement to count as step
max_ball_not_detected_frames = 20  # Frames before resetting step count
```

### Double dribble parameters

```python double_dribble.py
# Line 30-31: Tune hold detection
self.hold_duration = 0.85      # Seconds to consider ball as held
self.hold_threshold = 300      # Pixel distance from wrist to ball

# Line 41: Adjust dribble sensitivity
self.dribble_threshold = 18    # Vertical movement threshold
```

<Tip>
  Start with the default values and adjust based on your specific video quality, camera angle, and use case. Lower thresholds make detection more sensitive, while higher thresholds reduce false positives.
</Tip>

## Troubleshooting

### Basketball model not found

If you see an error about `basketballModel.pt` not being found:

1. Verify you downloaded the model from the Google Drive link
2. Ensure the file is in the project root directory
3. Check that the filename is exactly `basketballModel.pt`

### Webcam not working

If the webcam fails to open:

```python
# Try different camera indices
cap = cv2.VideoCapture(1)  # Instead of 0
```

Most systems use index 0 for the default camera, but some may require 1 or higher.

### Low detection accuracy

- Ensure good lighting conditions
- Position the camera to capture the full player and basketball
- Try adjusting confidence thresholds in the code:

```python
ball_model(frame, verbose=False, conf=0.5)  # Lower from 0.65
pose_model(frame, verbose=False, conf=0.3)  # Lower from 0.5
```

### Missing dependencies

If you encounter import errors:

```bash
pip install opencv-python numpy ultralytics gtts playsound
```
